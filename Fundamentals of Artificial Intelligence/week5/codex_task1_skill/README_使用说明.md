# 这个 skill 怎么用

## 你现在手里有什么
你目前给到我的材料说明：
- 项目是第 5 周 CLIP 图片打标签实践。
- 你们做的是 **任务 1：电商商品质检助手**。
- 代码模板已经是 CLIP 零样本分类模板。
- 你还需要把系统改成：**你电脑跑后端模型，另一台电脑做前端上传图片并接收结果**。

## 先纠正一个关键点
这个作业本质上**不一定需要训练模型**。
如果你们现在用的是 CLIP 零样本分类，那么更准确的说法是：
- 不训练传统分类器
- 而是优化标签词表
- 做测试集评估
- 封装成前后端分离的可演示系统

## 你应该怎么把它放进 Codex
### 方案 A：直接放进仓库根目录
把以下两个文件放进项目仓库根目录：
- `SKILL.md`
- `AGENTS.md`

然后把模板代码参考文件放进去，交给 Codex 执行。

### 方案 B：作为独立 skill 参考包
把本文件夹整体保存下来，之后在 Codex 中：
- 打开你的仓库
- 明确告诉 Codex 使用这个 skill 的规则
- 让它按 skill 改造 `week5/project_one`

## 建议你在 Codex 里直接输入的提示词
可直接复制这段：

```text
请使用仓库中的 SKILL.md 和 AGENTS.md，帮我完成 week5/project_one 的任务 1。
要求：
1. 保持 CLIP 零样本分类，不要误改成无关训练流程。
2. 按电商商品质检助手需求，把标签改成完好 / 外包装破损 / 空盒三类。
3. 把系统改造成前后端分离：我的电脑运行后端模型，另一台电脑通过前端上传图片并拿到结果。
4. 后端用 FastAPI，提供 /health 和 /predict。
5. 前端提供一个最小可运行页面。
6. 补 requirements.txt、README.md、评测脚本。
7. 最后告诉我如何在局域网两台电脑上联调。
```

## 你自己手动落地时的顺序
1. 先让单机版跑通
2. 再改成 FastAPI 后端
3. 再用 HTML 前端调用
4. 再做局域网访问
5. 最后做测试集评估与展示

## 两台电脑联调最关键的 4 件事
1. 后端必须监听 `0.0.0.0`
2. 前端必须填对后端 IP，例如 `http://192.168.1.10:8000`
3. 两台电脑必须在同一局域网
4. 你电脑系统防火墙要放行 8000 端口
